Les directives COMPONENTS Créer un composant

Ajoute un composant au projet Angular en cours

Syntaxe : ng generate component <nom-composant>

La commande doit être lancée dans le terminal à la racine du projet.
Elle génère un nouveau composant dans nom-projet/src/app/nom-composant
Le composant est un dossier constitué des fichiers suivants :

  1. nom-composant.component.ts : Fichier composant écrit en TypeScript
  2. nom-composant.component.html : Template du composant écrit en Html
  3. nom-composant.component.css : Fichier css spécifique à ce composant (vide par défaut)
  4. nom-composant.component.spec.ts : Fichier de specification pour les tests unitaires écrit en TypeScript
Structure du composant

Définit le selecteur du composant, le template html lié, et le fichier de style associé.

Syntaxe : @Component ({{'{'}} ... }) 

Fichier TypeScript nom-composant.component.ts
@Component ({{'{'}}
  selector : 'app-nom-composant',
  templateUrl : './app.component.html',
  styleUrls : ['./app.component.css']
})

L'annotation @Component permet de déclarer les trois éléments suivants :

Le selecteur css selector : 'app-nom-composant' permet à Angular d'identifier la balise propre au composant (<app-nom-composant></app-nom-composant>) sur le template html et de créer une instance de ce composant par l'intermédiaire de sa classe. C'est à partir de ces selecteurs que l'on va construire l'application depuis la page index.html

Le template html templateUrl : './monComposant.component.html'définit l'url du fichier html qui sera utilisée comme template. Le template est un morceau de code html qui pourra être inséré dans d'autres composants, ou bien comme c'est le cas pour le composant de base (app.component.html / app.component.ts / app.component.css) créé lors de l'initialisation d'un projet Angular, directement sur la page index.html.

Fichier html index.html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <title>Angular Beginner Project</title>
    <base href="/">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <link rel="icon" type="image/x-icon" href="favicon.ico">
  </head>
  <body>
    <app-component></app-component>
  </body>
</html>

Page racine, entrée dans l'application dans laquelle se trouve uniquement le composant de base qui regroupe tous les autres composants. 

Fichier html app-component.html
  <app-header></app-header>
  <app-main-content></app-main-content>
  <app-side-menu></app-side-menu>
  <app-footer></app-footer>

Template html du composant principal construit à partir de composants de mise en page.

Instanciation du composant

Le composant est instancié par l'intermédiaire de sa classe (TypeScript)

Fichier TypeScript nom-composant.component.ts
export class monComposant {{'{'}}
        attribut1 : string = 'valeur';
        attribut2 : number = 1337;
        constructor() {{'{'}}
        }
        methode1() : void {{'{'}}
          // Do something here
        }
        methode2(id : number) : boolean {{'{'}}
        return true;
        }
}

La classe du composant est écrite en TypeScript.

Elle peut contenir des attributs, un constructeur, des méthodes. Le cycle de vie des composants est géré par Angular, il n'est pas nécessaire de créer, détruire, modifier manuellement les instances des classes de nos composants.

Les méthodes sont accessibles via des évènements sur les templates. Les attributs peuvent être lus et affichés sur le template, et modifiés par l'intermédiaire de formulaires.

 STRUCTURAL DIRECTIVES ngIf

Ajoute ou retire conditionnellement un élément au DOM

Syntaxe : {{'*'}}ngIf = "condition"

Exemple :

Fichier TypeScript nom-composant.component.ts
export class monComposant {{'{'}}
   title : string = 'Angular Beginner Project';
   isTrue : boolean = false;
   toggleView() : void {{'{'}}
      this.isTrue = !this.isTrue;
   }
}
            
Fichier html nom-composant.component.html
<span *ngIf="isTrue"> Texte conditionnel <\span>
<button (click)="toggleView()"> {{isTrue ? 'Cacher' : 'Afficher'}} </button>

Rendu :

Texte conditionnel

ngFor

Permet d'itérer sur les éléments d'un tableau dans le DOM et d'ajouter des éléments HTML pour chaque itération

Syntaxe : {{'*'}}ngFor = "let item of list"

Exemple :

Fichier TypeScript app.component.ts
export class monComposant {{'{'}}
   title : string = 'Angular Beginner Project';
   isTrue : boolean = false;
   informations: string[] = [
     'L\'état qui engendre la règle est différent de celui que la règle engendre.',
     'Qui ne croit en lui-même, ment toujours.'
   ];
   toggleView() : void {{'{'}}
     this.isTrue = !this.isTrue;
   }
}



Fichier html app.component.html
<ul>
  <li *ngFor="let info of informations">{{'{{'}}info}}</li>
</ul>
Rendu :
  • {{info}}
 ngSwitch

Affichage conditionné à un test switch

Syntaxe :

{{'[ '}}ngSwitch ] = "variableATester" pour le conteneur.

*ngSwitchCase = " 'valeur' " pour les éléments conditionnels.

*ngSwitchDefault = " 'valeur' " pour les cas par défaut.

Attention aux simple et double quotes : si la valeur est écrite en dur il faut l'entourer de double quotes ET de simple quotes. Dans le cas d'une "variable" les double quotes suffisent.

   toggleView() : void {{'{'}}

   changeDay() : void {{'{'}}
     let randNum = Math.floor(Math.random()*11

       if(this.jour === 'vendredi') {{'{'}}
         this.jour=== 'lundi';
       }else {{'{'}}
         this.jour=== 'vendredi';
       }
     }else {{'{'}}

 ATTRIBUTE DIRECTIVES ngModel (Two-ways-Binding)

Permet de faire des liens bidirectionnels entre composant et template

Syntaxe : ([ ngModel ]) = "variable" et {{'{{'}}variable}} pour l'affichage

Exemple :

Fichier TypeScript app.component.ts

   title : string = 'Angular Beginner Project';
   isTrue : boolean = false;
   informations: string[] = [
     'L\'état qui engendre la règle est différent de celui que la règle engendre.',
     'Qui ne croit en lui-même, ment toujours.'
   ];
   jour: string = 'vendredi';
   variableDynamique: number = 0;
   toggleView() : void {{'{'}}
            this.isTrue = !this.isTrue;
   }
   changeDay() : void {{'{'}}
            let randNum = Math.floor(Math.random()*11
     if(randNum !== 10) {{'{'}}
            if(this.jour === 'vendredi') {{'{'}}
            this.jour=== 'lundi';
       }else {{'{'}}
            this.jour=== 'vendredi';
       }
     }else {{'{'}}
            this.jour=== '???';
   }
}

Fichier html app.component.html
<input type="number" ([ ngModel ]) = "variableDynamique"/>
  <p>
    J'ai {{variableDynamique}} an{{variableDynamique>1 ? 's' : ''}} d'expérience dans mon métier.
  </p>

Rendu :

J'ai {{variableDynamique}} an{{variableDynamique>1 ? 's' : ''}} d'expérience dans mon métier.

La condition ternaire ajoutée à l'affichage de variableDynamique permet de mettre 'an' au pluriel si sa valeur est supérieure à 1.

 ngClass

Changement dynamique de la classe d'un élément HTML en fonction de la valeur d'une variable définie dans le composant

Syntaxe : [ngClass] = "classesAUtiliser"

Exemple :

Fichier TypeScript app.component.ts
export class monComposant {{'{'}}
   title : string = 'Angular Beginner Project';
   isTrue : boolean = false;
   informations: string[] = [
     'L\'état qui engendre la règle est différent de celui que la règle engendre.',
     'Qui ne croit en lui-même, ment toujours.'
   ];
   jour: string = 'vendredi';
   isModified: boolean = false;
   classesAUtiliser: any = {{'{'}}};
   variableDynamique: number = 0;

   toggleView() : void {{'{'}}
     this.isTrue = !this.isTrue;
   }
   changeDay() : void {{'{'}}
     let randNum = Math.floor(Math.random()*11
     if(randNum !== 10) {{'{'}}
       if(this.jour === 'vendredi') {{'{'}}
          this.jour=== 'lundi';
       }else {{'{'}}
          this.jour=== 'vendredi';
       }
     }else {{'{'}}
       this.jour=== '???';
   }
   changerClasses() : void {{'{'}}
      this.classesAUtiliser = {{'{'}}
        normal : !this.isModified,
        modified : this.isModified,
      }
   }
   toggleModification() : void {{'{'}}
      this.isModified = !this.isModified;
      this.changerClasse();
   }
}

Fichier Css app.component.css

  font-weight: normal;
  color: black;
}

.modified {{'{'}}
  font-weight: bolder;
  text-shadow: white 1px 1px 4px;
  color: #20032a;
  font-size: 1.5em;
  border: 2px solid black;
  border-radius: 10px;
  padding: 0 10px;
  background: linear-gradient(90deg, cadetblue, lightsteelblue);
}
            
Fichier html app.component.html
<span [ngClass]="classesAUtiliser"/>
    Contenu changeant de classe
</span>
<button (click) ="toggleModification()"> Modifier le texte </button>

Rendu :

Contenu changeant de classe

Explication : La directive [ngClass] renvoie vers classeAUtiliser qui est un objet ayant pour clé les noms des classes à utiliser, et en valeur un booléen.

Il est possible d'utiliser une valeur en dur pour envoyer directement le nom d'une classe, ce qui n'a peu d'intérêt par rapport à l'utilisation classique de class="nomClasse" En envoyant un objet, on peut contrôler les classes qui seront affichées ou non, en contrôlant les valeurs booléennes associées aux clés.

C'est ce que nous faisons ici en utilisant deux classes dans notre objet classeAUtiliser, si les deux clés ont pour valeur true alors elles seront appliquées l'une après l'autre. Si certaines ont pour valeur false, elles ne seront pas appliquées avec [ngClass].

Dans l'exemple : le bouton permet de faire basculer isModified entre true et false, et d'appeler la méthode changerClasse qui va mettre à jour ses valeurs et donc l'affichage des classes de [ngClass].
