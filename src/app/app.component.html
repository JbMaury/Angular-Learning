Liaison unidirectionnelle

COMPOSANT -----> TEMPLATE HTML

Interpolation

Affiche sur le template la valeur d'une variable du composant

Syntaxe : {{'{{'}}variable}}

Exemple : <p> Le titre de mon application est : {{'{{'}}title}}</p> 

Fichier TypeScript app.component.ts
  export class monComposant {{'{'}}
          title :string = 'demo-angular';
          {{'}'}}

Rendu : Le titre de mon application est : {{title}}

Property Binding

Lit la valeur d'une variable du composant et l'ajoute en tant que valeur de propriété sur un élément HTML

Syntaxe : [ propriete ] = "variable"

Exemple : <input type="text" [value]="valeurInput"/> 

Fichier TypeScript app.component.ts
  export class monComposant {{'{'}}
          title :string = 'demo-angular';
          valeurInput : string = 'une valeur venant du composant';
          {{'}'}}
Le value de l'input est automatiquement rempli avec la valeur de "valeurInput" du composant.
TEMPLATE HTML -----> COMPOSANT

Event Binding

Envoie des données au composant depuis un évènement sur le template

Syntaxe : {{'('}}evenement) = "methode"

Exemple : <button {{'('}}click)="onclickButton()">Afficher infos</button> 

Fichier TypeScript app.component.ts
  export class monComposant {{'{'}}
            title :string = 'demo-angular';
            valeurInput : string = 'une valeur venant du composant';
            info :string = '';
            onclickButton() :void {{'{'}}
              if(this.info == ''){{'{'}}
                this.info = 'Une information très intéressante';
              }else {{'{'}}
                this.info = '';
              }
            }
          {{'}'}}

Rendu :

{{info}}

Liaison bidirectionnelle

COMPOSANT <-----> TEMPLATE HTML

Affiche et modifie dynamiquement la valeur de la variable du composant

Syntaxe : {{'[('}}ngModel)] = "variable" et {{'{{'}}variable}} pour l'affichage

Exemple : <input type="text" {{'[('}}ngModel)]="zoneSaisie"/> <p> {{'{{'}}zoneSaisie}} </p>

Les directives

COMPONENTS

Créer un composant

Ajoute un composant au projet Angular en cours

Syntaxe : ng generate component "nom-composant"

La commande doit être lancée dans le terminal à la racine du projet.
> Elle génère un nouveau composant dans src/app/nom-composant
> Le composant est constitué des fichiers suivants :

  1. nom-composant.component.ts : Fichier composant
  2. nom-composant.component.html : Template du composant
  3. nom-composant.component.css : Fichier css spécifique à ce composant
  4. nom-composant.component.spec.ts : Fichier de specification pour les tests

Structure du composant

Définit le selecteur css du composant et le template html lié

Syntaxe : @Component 

Fichier TypeScript nomComposant.component.ts
@Component({{'{'}}
  selector: 'app-nom-composant',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})

L'annotation @Component permet de déclarer les trois éléments suivants :

le selecteur css selector : 'app-nom-composant'

Permet à Angular d'identifier les balises propres au composant et de créer une instance de ce composant par l'intermédiaire de sa classe.

Le template html templateUrl: './monComposant.component.html'

Définit l'url de la page html qui sera utilisée comme template. Le template est un morceau de code Html qui pourra être inséré dans dans d'autres composants, ou bien comme c'est le cas pour le composant de base créé lors de l'initialisation d'un projet Angular, directement sur la page index.html. Cette insertion se fait grâce au selecteur css sous la forme :

<app-nom-composant></app-nom-composant>

Fichier html (index.html ou nom.component.html)

<html lang="en">
<head>
  <meta charset="utf-8">
  <title>DemoAngular</title>
  <base href="/">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <link rel="icon" type="image/x-icon" href="favicon.ico">
</head>
<body>
  <app-nom-composant></app-nom-composant>
</body>
</html>

le fichier css lié styleUrls : ['./nomComposant.component.css'

Permet de lier un fichier css spécifique pour le template de ce composant.

Instanciation du composant

Le composant est instancié par l'intermédiaire de sa classe (TypeScript)

Fichier TypeScript nomComposant.component.ts
export class PiedDePageComponent {{'{'}}
            attributDInstance : string ='';
            attributDInstance : number = 0;

            constructor() {{'{'}}

            }

            methodeDInstance() {{'{'}}
            //do something
            }
}

La classe du composant est écrite en TypeScript.

Elle peut contenir des attributs, un constructeur, des méthodes. Le cycle de vie des composants est géré par Angular, il n'est pas nécessaire de créer, détruire, modifier manuellement les instances des classes de nos composants.

Les méthodes sont accessibles via des évènements sur les templates. Les attributs peuvent être lus et affichés sur le template, et écrits par l'intermédiaire de formulaires.

STRUCTURAL DIRECTIVES

ngIf

Ajoute ou retire conditionnellement un élément au DOM

Syntaxe : {{'*'}}ngIf = "condition"

Exemple : <p*ngIf="isTrue">Texte conditionnel </p>
et le bouton <button (click)="toggleView()">{{'{{'}}isTrue ? 'Cacher' : 'Afficher'}}</button>
Affichable conditionnel avec ternaire (le nom du bouton change en fonction de la valeur de isTrue) 

Fichier TypeScript app.component.ts
  export class monComposant {{'{'}}
            title :string = 'demo-angular';
            isTrue : boolean = false;
            toggleView() : void {{'{'}}
              this.isTrue = !this.isTrue;
            }
          {{'}'}}

Rendu : Texte conditionnel

ngFor

Permet d'itérer sur les éléments d'une liste dans le DOM et d'ajouter des éléments HTML pour chaque itération

Syntaxe : {{'*'}}ngFor = "let item of list"

Exemple :
<ul>
<li*ngFor="let info of informations">{{'{{'}}info}} </li>
</ul> 

Fichier TypeScript app.component.ts
  export class monComposant {{'{'}}
        title :string = 'demo-angular';
        isTrue : boolean = false;
        informations: string[] = [
          'L\'état qui engendre la règle est différent de celui que la règle engendre.',
          'Qui ne croit en lui-même, ment toujours.'
        ];
        toggleView() : void {{'{'}}
          this.isTrue = !this.isTrue;
        }

      {{'}'}}
Rendu :
  • {{info}}

ngSwitch

Affichage conditionné à un test switch

Syntaxe : {{'['}}ngSwitch] = "variableATester"
et pour chaque case : *ngSwitchCase="'valeur'" + case par défaut : *ngSwitchDefault

Exemple :
<div[ngSwitch]="jour">
<h1 *ngSwitchCase="'vendredi'">Super c'est la fin de la semaine !</h1>
<h1 *ngSwitchCase="'lundi'">Le week-end est encore tellement loin...</h1>
<h1 *ngSwitchDefault>Demain est un autre jour !</h1>
</div>
